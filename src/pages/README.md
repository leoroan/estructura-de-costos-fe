# Pages

## Propósito
Representar vistas o pantallas de alto nivel del producto.

## Cuándo usarlo
- Rutas principales de la aplicación.
- Pantallas completas como Home, Login, Dashboard o Settings.

## Reglas
- Componer páginas usando componentes y hooks.
- Evitar acoplar demasiada lógica de negocio directamente aquí.
- Mantener la pantalla orientada al flujo de usuario.
