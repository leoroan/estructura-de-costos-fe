# Styles

## Propósito
Centralizar estilos compartidos, temas y utilidades visuales.

## Cuándo usarlo
- Estilos globales.
- Clases base o utilidades de diseño.
- Tailwind config, theme tokens o componentes visuales compartidos.

## Reglas
- Mantener la capa visual consistente.
- Preferir utilidades y tokens reutilizables.
- Evitar estilos inline dispersos en componentes.
