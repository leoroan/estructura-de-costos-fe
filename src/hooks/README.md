# Hooks

## Propósito
Centralizar lógica reutilizable relacionada con estado, efectos o comportamiento compartido.

## Cuándo usarlo
- Lógica que se repite entre componentes.
- Manejo de efectos, suscripciones o integraciones con APIs.

## Reglas
- Mantenerlos simples y con una responsabilidad clara.
- No mezclar lógica de UI muy específica con lógica de negocio.
- Preferir hooks pequeños y composables.
